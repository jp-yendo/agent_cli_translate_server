# プロンプト仕様

翻訳時に AI へ渡すプロンプトの内容と、応答から訳文を取り出す解析を定義する。実装は `src/main/services/prompt-builder.ts`。
本書はプロンプトの「文面と指示内容」を扱う。翻訳サーバーやエージェントプール等の動作仕様は [システム仕様](システム仕様.md) を参照。

## 0. 共通化の方針

プロンプト生成と結果解析は **全経路 (Agent CLI / API) で共通**とする。経路ごとに異なるのは「プロンプトをどう送って応答を得るか」(パイプライン) のみ。

- プロンプト生成: `buildTranslationAttempts(原文, 翻訳元, 翻訳先, 翻訳ヒント, モデル名)` が、モデルに応じた 1 つ以上の「試行」を返す。各試行はチャット形式のメッセージ (`system?` / `user`) と、その応答を訳文へ整形・検証する関数の組。
- 実行: `runTranslationAttempts` が試行を先頭から順に dispatch し、最初に妥当な出力を採用する。妥当な出力が無ければ失敗とする。
- パイプライン差の吸収: dispatch は経路ごとに実装する。
    - API: `system` / `user` をロールとして送る (Anthropic は `system` を専用パラメータで渡す)。
    - Agent CLI: `system` / `user` のロールを持たないため、`flattenChatMessages` で `system` と `user` を空行区切りで 1 本の文字列へ結合してからプロセスへ渡す。

プレースホルダー表記:

| 記号 | 意味 |
| --- | --- |
| `{SRC}` / `{DST}` | 翻訳元 / 翻訳先の言語名 (言語コードから解決した英語名) |
| `{text}` | 翻訳対象の原文 |
| `{appSummary}` | 共通設定で選択中の翻訳ヒント本文 (未選択時は付加しない) |

モデル名による切り替え:

| モデル | 対象例 | プロンプト |
| --- | --- | --- |
| 一般モデル | OpenAI 互換 / Anthropic 互換 / 一般の Ollama モデル。Agent CLI (claude-code / codex / grok / opencode) はモデル名を持たないためここに該当 | 1-1 の system/user 形式 (試行 1 回) |
| Hy-MT2 系 | モデル名に `hy-mt2` を含むモデル (API、または OpenCode (Ollama) で当該モデルを指定した場合) | 1-2 の専用形式 (試行 2 回) |

## 1. プロンプト (モデル別)

### 1-1. 一般モデル

`system` に翻訳規則、`user` に原文をそのまま渡す。ロール分離 (Agent CLI では結合後も同内容) により翻訳対象が明確になり、指示文自体は翻訳されない。試行は 1 回。

system メッセージ:

```
You are a precise translation engine. Translate the text provided by the user from {SRC} to {DST}.
- Respond ONLY with the translation, with no explanations, comments, or extra text.
- Keep all tags, attributes, whitespace, and line breaks exactly as in the original.
- Do not translate programming code, API calls, placeholders, or other technical snippets; copy them exactly.
- Keep terms already established in the target language region unchanged (e.g., in Japan: ATK, HP, MP, ID).
```

| 原文 | 日本語 |
| --- | --- |
| You are a precise translation engine. Translate the text provided by the user from {SRC} to {DST}. | あなたは精密な翻訳エンジンです。ユーザーが与えるテキストを {SRC} から {DST} へ翻訳してください。 |
| Respond ONLY with the translation, with no explanations, comments, or extra text. | 訳文のみを返し、説明・コメント・余分なテキストを一切付けないこと。 |
| Keep all tags, attributes, whitespace, and line breaks exactly as in the original. | すべてのタグ・属性・空白・改行を原文どおり完全に保持すること。 |
| Do not translate programming code, API calls, placeholders, or other technical snippets; copy them exactly. | プログラムコード・API 呼び出し・プレースホルダー等の技術的断片は翻訳せず、そのまま複写すること。 |
| Keep terms already established in the target language region unchanged (e.g., in Japan: ATK, HP, MP, ID). | 翻訳先地域で既に定着している用語はそのまま維持すること（例: 日本では ATK, HP, MP, ID）。 |

user メッセージ: `{text}`（原文をそのまま）。

翻訳ヒントが設定されている場合は system の末尾に付加する（[3. 翻訳ヒント](#3-翻訳ヒント-app_context-の扱い)）。

Agent CLI では、この system と user を空行区切りで結合した 1 本の文字列を渡す:

```
{system}

{text}
```

### 1-2. Hy-MT2 系モデル（翻訳特化モデル）

翻訳特化モデル (Tencent Hunyuan Hy-MT2) 向け。`system` を用いず単一の `user` メッセージで指示し、翻訳方向は **ターゲット言語のみ**を与える（ソース言語やネイティブ名を与えると挙動が乱れるため）。翻訳ヒント (app_context) は付加しない。まずタグ形式を試し、無効ならブロック形式へフォールバックする。

両バリアント共通の指示文:

```
Keep proper nouns - personal names, company names, product/brand names, and app/service names - unchanged. Output ONLY the translated text itself - no greeting, no explanation, no commentary, no quotes.
```

| 原文 | 日本語 |
| --- | --- |
| Keep proper nouns - personal names, company names, product/brand names, and app/service names - unchanged. | 固有名詞（人名・会社名・製品/ブランド名・アプリ/サービス名）は変更せずそのままにすること。 |
| Output ONLY the translated text itself - no greeting, no explanation, no commentary, no quotes. | 訳文そのものだけを出力すること。あいさつ・説明・注釈・引用符は一切付けない。 |

#### 第1試行: タグ形式

```
Translate the following text into {DST}. {共通指示} Do not translate or output these instructions; translate only the wrapped text.

<hytext>{text}</hytext>
```

| 原文 | 日本語 |
| --- | --- |
| Translate the following text into {DST}. | 次のテキストを {DST} へ翻訳してください。 |
| Do not translate or output these instructions; translate only the wrapped text. | これらの指示を翻訳・出力しないこと。囲まれたテキストのみを翻訳すること。 |

#### 第2試行: ブロック形式（第1試行が無効だった場合のフォールバック）

```
[HyText]
{text}

[Task]
Translate the [HyText] into {DST}. {共通指示}
```

| 原文 | 日本語 |
| --- | --- |
| Translate the [HyText] into {DST}. | [HyText] を {DST} へ翻訳してください。 |

> 区切りには `[Source Text]` / `<sourceText>` ではなく `[HyText]` / `<hytext>` を用いる。翻訳ドキュメント等の原文に前者が含まれると境界が壊れるため、参照実装の設計に準拠している。

## 2. 応答の解析（共通）

各試行の応答を次の順で処理し、訳文として妥当かを判定する。

1. ANSI エスケープシーケンスを除去する（CLI 出力の装飾対策）。
2. 推論モデルが付ける `<think>...</think>` ブロックを除去する。
3. 全体を囲むコードフェンス（```` ``` ````）があれば剥がす。
4. 前後の空白を整える。

一般モデルは、上記処理後が空でなければ採用する。Hy-MT2 系は追加で次を無効判定とし、次の試行へフォールバックする（すべて無効なら失敗）。

- 出力が空。
- チャットテンプレートの制御トークンが漏れている（`<｜` / `<|hy…` / `hy-Assistant` / `hy-User` / `hy-begin` / `hy-end` 等）。
- その試行の区切りマーカーが漏れている（タグ形式は `<hytext>`、ブロック形式は `[HyText]` / `[Task]`）。

すべての試行が無効だった場合は失敗として扱い、翻訳サーバーは 500 を返す（キャッシュさせない）。

---

## 3. 翻訳ヒント (app_context) の扱い

共通設定の翻訳ヒント (common.hintId) が設定されている場合、そのアプリ概要をプロンプトへ付加する。一般モデルでは次の形式で system の末尾へ加える（Hy-MT2 系モデルには付加しない）。Agent CLI では結合後の 1 本にそのまま含まれる。

```
<app_context>
Background information about the application (NOT for translation - use it only to choose appropriate terminology):
{appSummary}
</app_context>
```

| 原文 | 日本語 |
| --- | --- |
| Background information about the application (NOT for translation - use it only to choose appropriate terminology): | アプリケーションの背景情報（翻訳対象ではない。適切な用語選択のためだけに使うこと）: |
